# MiMo Code 本地能力边界

MiMo Code 的手机端以轻量、可控的代码协作为目标。聊天界面可以用于分析代码、整理修改方案、解释错误、生成补丁建议和检查风险，但不在 Android 本地复制完整桌面开发环境。

## 本地范围

- 代码、日志、错误信息和需求分析
- 修改方案、代码片段和补丁建议
- 对话历史、角色、模型和上下文管理
- 复制、编辑、重试和重新生成
- 后续接入文件能力时，优先采用明确授权和变更预览

## 不在本地范围

- Gradle、Maven、Node、Python、Go、Rust 等完整构建工具链
- APK、AAB、桌面安装包或容器镜像打包
- 应用商店上传、服务器部署和生产发布
- 任意 Bash、PowerShell 或不受限制的系统命令
- 下载并直接执行未知二进制文件

## 设计原则

1. 会话和任务是界面主线，参考 Happy 的 session-first 信息架构。
2. 模型、角色和运行状态集中在顶部栏与输入面板，不增加长期占用空间的底部导航。
3. 助手回复使用开放式内容布局，给代码、说明和状态留出足够空间。
4. 高风险能力必须独立授权，不把打包、部署或发布伪装成普通聊天操作。
